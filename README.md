# Brain Cabinet v2

自分の考えを書き留めて、それを鏡のように映し返してくれる道具。

日記でもTODOリストでもナレッジベースでもない。
「自分がどう考えているか」に気づくための仕組み。

## なにをするのか

やることは3つだけ。

**書く** — 思ったことをそのまま書く。システムが「これは何の種類の思考か」「どれくらい確かか」を静かに判定する。AIに聞いたりはしない。すべてローカルで動く。

**気づく** — 書き溜めた思考の傾向が変わったとき、それを検知する。昨日まで技術のことばかり考えていたのに、今週は人間関係の話ばかりしている。そういう変化を「ドリフト」として浮かび上がらせる。鏡が黙ったら、それはただの壁掛けの絵と同じだから。

**戻る** — 過去に書いた思考を、忘れた頃にもう一度見せる。間隔反復（SM-2）のアルゴリズムで、ちょうどいいタイミングで再会させる。そのとき自分がどう感じるか——同意するのか、違和感があるのか——それが判断力を育てる。

この3つが回り続けることで、「書く→気づく→直す→また書く」のループができる。

## なにを使っているか

| 何を | 何で |
|------|------|
| API | Hono (Node.js) |
| データベース | SQLite + Drizzle ORM |
| テキストの意味理解 | MiniLM（ローカル、384次元ベクトル） |
| 画面 | React (Vite) |
| 定期処理 | node-cron |

外部AIサービスへの依存はゼロ。すべて手元で完結する。

## 動かし方

```bash
# 準備
npm install && cd ui && npm install && cd ..

# データベースを作る
npm run db:generate
npm run db:migrate

# 起動（バックエンドとフロントエンドが同時に立ち上がる）
npm run dev
```

バックエンドは `localhost:3000`、フロントエンドは `localhost:5173` で動く。

## 設計の考え方

APIはひとつだけ。`POST /api/v1` に `{ action, payload }` を投げる。アクションは9つ。テーブルは5つ。それで全部。

v1では38テーブル・100以上のアクションに膨らんだ。v2はそこから93%削ぎ落として、ループを回すのに本当に必要なものだけを残した。

「これがないとループが回らない」——その問いに答えられないものは入れない。
